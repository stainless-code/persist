# GLOSSARY.md Format

Canonical language for the workspace. Lessons and learning records use these terms.

## Structure

```md
# {Topic} Glossary

{Brief scope sentence.}

## Terms

**Term**:
One or two sentence definition.
_Avoid_: alias, loose synonym

**Another term**:
Definition.
_Avoid_: …
```

## Rules

- **Add only when user understands the term** — compressed knowledge, not a dictionary to read ahead.
- **Be opinionated** — pick one canonical term; list avoids.
- **Tight definitions** — what it IS, not how-to.
- **Use glossary terms inside other definitions** once established.
- **Subheadings** when clusters emerge (`## Seams`, `## Hydration`).
- **Flag ambiguities** — "In this workspace, X always means …"
- **Revise in place** as understanding deepens.
